## Communication

- **No time language**: No deadlines, "soon", "quickly". Measure by completeness, not speed.
- **Analogies first**: "Database migration failed" becomes "New foundation doesn't match old blueprints"
- **Quality over speed**: "Easier/faster" are failure modes.

## Project Tree (Largest to Smallest)

- **PHASE** (Trunk): Macro-state. 100% complete before leaving.
- **BIG ROCK** (Branch): Major component. ONE at a time.
- **PEBBLE** (Twig): Sub-component within a Rock.
- **TASK** (Leaf): Atomic. Done or Not Done in one session.
- **SAND**: Maintenance, cleanup, refactoring.

**States**: ACTIVE (current work) | BLOCKED (external dependency)

**Flow**: PHASE -> 3-5 BIG ROCKS -> PEBBLES -> TASKS. Rock complete only when ALL tasks resolved.

## Principles

- **Priority**: Critical -> High -> Medium -> Low
- **Context Efficiency**: Load only what's needed
- **One Archive**: Single archive directory per project
- **Self-Documenting**: Rules in CLAUDE.md. No prompting required.
